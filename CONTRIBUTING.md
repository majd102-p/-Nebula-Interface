# Contributing to Nebula Interface

Thanks for improving Nebula Interface.

## What this project expects

- Keep changes focused and easy to review.
- Prefer deterministic behavior over clever but fragile logic.
- Add tests or runtime checks when you change gesture logic, camera flow, or MQTT behavior.
- Update the README or the relevant guide when you change user-facing behavior.

## Before you open a pull request

1. Run the system test.
2. Verify the app starts and shuts down cleanly.
3. Check the documentation for stale instructions.
4. Keep markdown and code formatting clean.

## Suggested PR structure

- One meaningful feature or fix per pull request.
- Include a short summary of the behavior change.
- Mention any config keys you added or changed.
- Add screenshots or clips if the HUD or workflow changed.

## Good contribution areas

- Gesture reliability and false-positive reduction
- Calibration and prototype refinement
- Runtime diagnostics and telemetry
- Documentation and quick-start flow
- UI or HUD clarity

## Review standard

Changes should be safe, understandable, and reproducible.
